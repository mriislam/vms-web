🚗 NEXVMS · Fleet Management System

API Reference

RESTful JSON API for integrating with NEXVMS — covering vehicle booking, fleet tracking, driver mobile apps, geofencing, and operations management.

Base URL https://demo-vms.nexdecade.com/api
Auth Bearer JWT (HS512)
Encryption AES-256-GCM (optional)
Content-Type application/json
Authentication

Bearer Token

All protected endpoints require a JWT in the Authorization header. Obtain a token via /auth/login. Tokens expire after 24 hours.

# Include on every protected request
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...
ℹ Endpoints under /api/auth/login, /api/auth/verify-mfa, and /api/health are public — no token required.
Payload Encryption

AES-256-GCM

All non-auth API request and response bodies are encrypted. Auth endpoints (/api/auth/*) and the health endpoint are sent as plain JSON.

Key Derivation

// Shared secret
const passphrase = "VMS-Nexdecade-AES256-SecretKey-2024"
const keyBytes   = SHA256(passphrase)  // 32-byte key

// Encrypt (AES-GCM, 12-byte random IV)
const iv         = randomBytes(12)
const ciphertext = AES_GCM_encrypt(key, iv, JSON.stringify(payload))
const combined   = concat(iv, ciphertext, gcmTag)   // IV(12) + CT + Tag(16)

// Send as JSON wrapper
{ "enc": base64(combined) }

// Decrypt response
const data = base64decode(response.enc)
const iv   = data.slice(0, 12)
const plain = AES_GCM_decrypt(key, iv, data.slice(12))
Error Handling

Response Format

{
  "success": true | false,
  "message": "Human-readable message",
  "data": { /* payload or null */ }
}
StatusMeaning
200Success — data in data field
400Bad request — validation failed or wrong input
401Unauthorized — missing or invalid JWT
404Resource not found
500Server error — check backend logs
Auth Endpoints
POST /auth/login Authenticate and obtain JWT

Request Body

FieldTypeRequiredDescription
usernamestringrequiredAccount username
passwordstringrequiredAccount password

Responses

200No 2FA — returns JWT token directly
2002FA enabled — returns mfaRequired: true + mfaToken
// No 2FA response
{ "data": { "token": "eyJ...", "username": "admin", "fullName": "System Admin", "role": "admin" } }

// 2FA required response
{ "data": { "mfaRequired": true, "mfaToken": "uuid-xxx", "username": "admin" } }
POST /auth/verify-mfa Verify TOTP code — step 2 of 2FA login 
{ "mfaToken": "uuid from login", "code": "123456" }
200Returns JWT token (same as no-2FA login response)
400Wrong code — user can retry (token not consumed on failure)
GET /auth/setup-mfa 🔒 JWT Generate 2FA secret + QR code URI for logged-in user 
{ "secret": "BASE32SECRET", "otpUri": "otpauth://totp/NEXVMS:user?secret=...", "account": "admin" }
💡Render otpUri as a QR code for Google Authenticator to scan.
POST /auth/enable-mfa 🔒 JWT Confirm and activate 2FA after scanning QR 
{ "code": "123456" }  // First code from Authenticator app
POST /auth/disable-mfa 🔒 JWT Disable 2FA (user enters their own code to confirm) 
{ "code": "123456" }
POST /auth/reset-mfa/{username} 🔒 Admin Admin disables 2FA for any user

No request body. Resets mfa_enabled=false and clears the secret for the specified user.
